기초 다지기: 프로그래밍의 첫걸음

모든 것의 시작은 기초입니다. 프로그래밍이 무엇인지, 그리고 우리가 배우는 파이썬이 어떤 방식으로 작동하는지 이해하는 것은 매우 중요합니다. 이 섹션에서는 가장 기본적인 개념들을 퀴즈로 풀어보며 확실히 짚고 넘어갑니다.

기초 개념 퀴즈

1. 다음 중 '프로그램'과 '프로그래밍'에 대한 설명으로 옳은 것은 무엇인가요?

해설:

프로그램은 컴퓨터가 이해할 수 있는 '명령어의 모음'(결과물)이고, 프로그래밍은 그 프로그램을 '만드는 과정'(행위)입니다.

"헷갈리지 마요~ 프로그램은 결과, 프로그래밍은 만드는 과정이에요~"

2. 파이썬은 어떤 방식의 언어인가요?

해설:

파이썬은 인터프리터 언어입니다. 코드를 한 줄씩 바로 번역하고 실행해서 결과를 즉시 확인할 수 있죠. 마치 동시통역사 같아요!

"파이썬은 인터프리터예요! 즉, 바로바로 실행 가능하다는 말이에요~"

변수와 자료형: 데이터 담기

변수는 데이터를 저장하는 '이름 붙인 상자'입니다. 어떤 종류의 데이터를 담느냐에 따라 자료형이 결정되죠. 아래 입력창에 여러 값을 넣어보며 파이썬이 어떻게 자료형을 인식하는지 직접 확인하고, 변수 이름 규칙도 맞춰보세요.

자료형 탐지기

변수명 규칙 퀴즈

1. 다음 중 파이썬에서 올바른 변수명은 무엇인가요?

해설:

  • 변수 이름은 반드시 영문자 또는 밑줄(`_`)로 시작해야 합니다. (숫자로 시작 불가)
  • 특수문자(`+`, `-` 등), 띄어쓰기는 사용할 수 없습니다.
  • `while`과 같은 파이썬의 예약어는 변수명으로 사용할 수 없습니다.

"숫자로 시작하면? 파이썬은 계산식인 줄 안다니까~"

연산자 마스터: 계산의 달인

연산자는 더하기, 빼기 같은 계산을 수행하는 기호입니다. 파이썬에는 독특한 연산자들이 많아요. 직접 숫자를 넣고 연산자를 클릭하며 그 차이를 느껴보고, 문자열 연산 퀴즈도 풀어보세요.

파이썬 계산기

"슬래시 두 번! 퍼센트! 별표 두 개! 이건 진짜 많이 나와요!"

문자열 연산 퀴즈

1. 다음 코드의 실행 결과로 올바른 것은 무엇인가요?

a = "파이썬"
b = " 최고"
print(a * 2 + b)

해설:

파이썬은 연산자 우선순위에 따라 곱셈(`*`)을 먼저 처리한 후 덧셈(`+`)을 수행합니다. 따라서 `a * 2`가 먼저 '파이썬파이썬'이 되고, 그 뒤에 `b`(" 최고")가 연결됩니다.

"문자 * 숫자? 돼요! 세 번 반복되게! 근데 문자 + 숫자는 안 돼요~ 에러나요!"

파일 입출력 퀴즈

1. 파일 입출력(File I/O) 작업의 올바른 순서는 무엇인가요?

해설:

파일을 사용하기 위해서는 반드시 '열기' → '처리(읽기/쓰기)' → '닫기' 순서를 지켜야 합니다. 파일을 닫지 않으면 내용이 저장되지 않을 수 있습니다.

"파일은 읽고 쓰고 닫아야 돼요. 까먹으면 저장이 안 돼요!"

리스트 정복: 여러 데이터 관리

데이터가 많아지면 변수 하나로는 부족해요. 리스트를 사용하면 여러 개의 데이터를 하나의 묶음으로 효율적으로 관리할 수 있습니다. 1차원 리스트부터 행과 열 구조를 가진 2차원 리스트까지, 직접 조작해보며 마스터해봅시다.

리스트 시각화 및 조작 도구

"인덱스는 무조건 0부터 시작! 2차원 리스트는 진짜 중요해요~"

리스트 퀴즈

1. 다음 2차원 리스트 `menu`에서 '튀김'의 가격인 2000을 출력하기 위한 코드로 올바른 것은 무엇인가요?

menu = [["라볶이", 4000], ["순대", 3000], ["튀김", 2000]]

해설:

리스트의 인덱스는 0부터 시작합니다. `menu[2]`는 `["튀김", 2000]` 리스트를 가리키고, 그 리스트의 두 번째 요소인 인덱스 `[1]`이 가격 2000을 정확히 선택합니다.

"여기서 실기 나오면, 인덱스 하나라도 헷갈리면 틀려요~ 조심!"

제어 구조: 프로그램의 흐름 만들기

프로그램을 똑똑하게 만들려면 상황에 따라 다른 행동을 하거나, 특정 작업을 반복하게 해야 합니다. `if`문과 `while`문은 프로그램의 흐름을 제어하는 가장 중요한 도구입니다. 이 개념은 앞으로 배울 모든 것의 기초가 됩니다.

조건문 (if)

'만약 ~라면'처럼 특정 조건이 맞을 때만 코드를 실행합니다.

score = 95
if score >= 90:
    print("A학점 입니다!")
elif score >= 80:
    print("B학점 입니다.")
else:
    print("노력이 필요해요.")

위 코드의 실행 결과는 무엇인가요?

해설:

`score`가 95이므로 첫 번째 조건 `score >= 90`이 참(True)이 되어 "A학점 입니다!"가 출력됩니다.

반복문 (while)

'~하는 동안'처럼 특정 조건이 만족되는 동안 코드를 계속 반복합니다.

count = 0
while count < 3:
    print("반복!")
    count += 1

위 코드는 '반복!'을 몇 번 출력할까요?

해설:

`count`가 0, 1, 2일 때 조건 `count < 3`이 참이므로 '반복!'을 출력하고 `count`가 1씩 증가합니다. `count`가 3이 되면 조건이 거짓이 되어 반복이 종료됩니다. 따라서 총 3번 출력됩니다.

"제어 구조가 문법에서는 가장 핵심이 될 것 같아. 이거 헷갈리면 뒤에 다 꼬여요~"

✨ 최종 정리: 수행평가 완벽 대비

지금까지 배운 내용들을 바탕으로 수행평가에서 나올 수 있는 핵심적인 지필 및 프로그래밍 문제 유형을 최종 정리해 드립니다. 각 포인트를 다시 한번 짚어보며 시험에 완벽하게 대비하세요!

지필 평가 예상 문제 유형

  • 변수와 리스트의 차이점 및 사용 목적 설명: 데이터를 한 개 저장하는 변수와 여러 값을 저장하는 리스트의 개념, 그리고 왜 리스트가 필요한지 설명할 수 있어야 합니다.
  • 파이썬의 특징과 인터프리터 방식 설명: 파이썬이 문법이 간단하고 활용성이 높은 이유, 그리고 코드를 한 줄씩 번역/실행하는 인터프리터 방식의 특징을 설명할 수 있어야 합니다.
  • 문자열 연산과 형 변환의 중요성 설명: 문자열 덧셈(`+`), 곱셈(`*`)의 특징, 그리고 `input()` 함수로 받은 문자열을 `int()`나 `float()`으로 형 변환해야 하는 이유를 설명할 수 있어야 합니다.
  • 파일 입출력의 3단계 설명: 파일 열기, 처리, 닫기의 중요성과 각 단계의 의미를 설명할 수 있어야 합니다.

프로그래밍 실기 예상 문제 유형

  • 2차원 리스트 활용: 주어진 2차원 리스트에서 특정 요소(예: 메뉴의 가격)를 인덱싱하여 출력하는 코드를 작성할 수 있어야 합니다. 변수명 스펠링 주의!
  • 사용자 입력, 연산, 형 변환 활용: `input()` 함수로 사용자로부터 값을 입력받고, `int()` 또는 `float()`로 형 변환하여 사칙연산(`+`, `*`, `//`, `%`, `**`)을 수행한 후 결과를 출력하는 코드를 작성할 수 있어야 합니다. (예: BMI 계산기, 과목 평균 구하기)
  • 파일 입출력 코딩: 파일에 특정 내용을 쓰고(write), 닫은(close) 후, 다시 파일을 열어(open) 내용을 읽어(read) 화면에 출력하는 코드를 작성할 수 있어야 합니다.
  • 리스트 조작 (추가/삭제/수정): 리스트에 요소를 추가(`append`), 삭제(`remove`), 특정 인덱스의 값을 수정하는 코드를 작성할 수 있어야 합니다.
  • 제어 구조 활용: `if`문(조건문)과 `while`문(반복문)을 활용하여 특정 조건에 따라 다른 코드를 실행하거나, 반복 작업을 수행하는 간단한 코드를 작성할 수 있어야 합니다. (예: 점수에 따른 학점 출력, 반복 횟수 제어)
  • 2차원 리스트로 이미지 표현 (픽셀 아트): 0과 1로 구성된 2차원 리스트를 사용하여 간단한 이미지(예: 하트)를 표현하는 코드를 이해하고 작성할 수 있어야 합니다.
