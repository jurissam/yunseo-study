기초 다지기: 프로그래밍의 첫걸음

모든 것의 시작은 기초입니다. 프로그래밍이 무엇인지, 그리고 우리가 배우는 파이썬이 어떤 방식으로 작동하는지 이해하는 것은 매우 중요합니다. 이 섹션에서는 가장 기본적인 개념들을 퀴즈로 풀어보며 확실히 짚고 넘어갑니다.

기초 개념 퀴즈

1. 다음 중 '프로그램'과 '프로그래밍'에 대한 설명으로 옳은 것은 무엇인가요?

해설:

프로그램은 컴퓨터가 이해할 수 있는 '명령어의 모음'(결과물)이고, 프로그래밍은 그 프로그램을 '만드는 과정'(행위)입니다.

"헷갈리지 마요~ 프로그램은 결과, 프로그래밍은 만드는 과정이에요~"

2. 파이썬은 어떤 방식의 언어인가요?

해설:

파이썬은 인터프리터 언어입니다. 코드를 한 줄씩 바로 번역하고 실행해서 결과를 즉시 확인할 수 있죠. 마치 동시통역사 같아요!

"파이썬은 인터프리터예요! 즉, 바로바로 실행 가능하다는 말이에요~"

3. 파이썬 코드에서 `#` 기호의 역할은 무엇인가요?

해설:

`#` 기호 뒤에 오는 내용은 파이썬 인터프리터가 무시하며, 코드에 대한 설명이나 메모를 남길 때 사용됩니다. 이를 '주석'이라고 합니다. 프로그램 실행에는 영향을 주지 않아요.

"주석은 코드를 설명하는 나만의 메모! 컴퓨터는 안 읽어요~"

4. 다음 중 컴퓨터 과학에서 '알고리즘'에 대한 설명으로 가장 적절한 것은 무엇인가요?

해설:

알고리즘은 어떤 문제를 해결하기 위한 '명확하고 유한한 절차'를 의미합니다. 컴퓨터 프로그램은 이러한 알고리즘을 코드로 구현한 것입니다.

"알고리즘은 문제 해결을 위한 요리 레시피 같은 거예요~ 순서가 중요하죠!"

5. 다음 중 '버그(Bug)'의 의미로 가장 올바른 것은?

해설:

'버그'는 컴퓨터 프로그램의 오류나 결함을 의미합니다. 버그 때문에 프로그램이 멈추거나, 원치 않는 결과를 내거나, 아예 실행되지 않을 수도 있습니다.

"버그 잡는 걸 디버깅이라고 해요! 이거 없애야 프로그램이 똑똑해져요!"

6. 파이썬으로 작성된 코드를 컴퓨터가 이해하고 실행할 수 있도록 변환하는 역할을 하는 것은 무엇인가요?

해설:

인터프리터는 파이썬 코드를 한 줄씩 읽어서 컴퓨터가 이해하는 기계어로 변환하고 바로 실행합니다. 이 과정 덕분에 파이썬은 작성과 동시에 결과를 확인할 수 있습니다.

"인터프리터는 코드 통역사! 없으면 컴퓨터는 우리말 못 알아들어요~"

변수와 자료형: 데이터 담기

변수는 데이터를 저장하는 '이름 붙인 상자'입니다. 어떤 종류의 데이터를 담느냐에 따라 자료형이 결정되죠. 아래 입력창에 여러 값을 넣어보며 파이썬이 어떻게 자료형을 인식하는지 직접 확인하고, 변수 이름 규칙도 맞춰보세요.

자료형 탐지기

변수명 규칙 퀴즈

1. 다음 중 파이썬에서 올바른 변수명은 무엇인가요?

해설:

  • 변수 이름은 반드시 영문자 또는 밑줄(`_`)로 시작해야 합니다. (숫자로 시작 불가)
  • 특수문자(`+`, `-` 등), 띄어쓰기는 사용할 수 없습니다.
  • `while`과 같은 파이썬의 예약어는 변수명으로 사용할 수 없습니다.

"숫자로 시작하면? 파이썬은 계산식인 줄 안다니까~"

2. 파이썬은 변수를 선언할 때 자료형을 명시해야 하나요? (예: `int x = 10;`)

해설:

파이썬은 '동적 자료형(Dynamic Typing)'을 지원하는 언어입니다. 이는 변수를 선언할 때 자료형을 미리 지정할 필요 없이, 값이 할당될 때 자동으로 자료형이 결정된다는 의미입니다.

"파이썬은 스마트해서 자료형을 알아서 맞춰줘요! 우리는 그냥 쓰면 돼요~"

3. 다음 파이썬 코드에서 `type(x)`의 결과는 무엇인가요?

x = 3.14

해설:

소수점이 있는 숫자는 파이썬에서 '실수(float)' 자료형으로 인식됩니다. `type()` 함수는 변수의 자료형을 알려줍니다.

"점 찍으면 다 실수! 자료형 확인은 type()이야~"

4. 다음 중 파이썬에서 문자열을 나타내는 방법으로 올바르지 않은 것은 무엇인가요?

해설:

파이썬에서 문자열은 작은따옴표 (`'`), 큰따옴표 (`"`), 또는 세 개의 따옴표 (`"""` 또는 `'''`)로 둘러싸여야 합니다. 대괄호 `[]`는 리스트를 나타낼 때 사용됩니다.

"문자열은 따옴표 싸야 돼! 대괄호는 리스트 거라고!"

5. 다음 파이썬 코드에서 변수 `is_active`의 자료형은 무엇인가요?

is_active = True

해설:

`True`와 `False`는 참/거짓을 나타내는 '불(Boolean)' 자료형입니다. 논리적인 조건을 다룰 때 사용됩니다.

"True, False는 불리언! T랑 F는 꼭 대문자로 시작해야 해!"

연산자 마스터: 계산의 달인

연산자는 더하기, 빼기 같은 계산을 수행하는 기호입니다. 파이썬에는 독특한 연산자들이 많아요. 직접 숫자를 넣고 연산자를 클릭하며 그 차이를 느껴보고, 문자열 연산 퀴즈도 풀어보세요.

파이썬 계산기

"슬래시 두 번! 퍼센트! 별표 두 개! 이건 진짜 많이 나와요!"

연산자 퀴즈

1. 다음 코드의 실행 결과로 올바른 것은 무엇인가요?

a = "파이썬"
b = " 최고"
print(a * 2 + b)

해설:

파이썬은 연산자 우선순위에 따라 곱셈(`*`)을 먼저 처리한 후 덧셈(`+`)을 수행합니다. 따라서 `a * 2`가 먼저 '파이썬파이썬'이 되고, 그 뒤에 `b`(" 최고")가 연결됩니다.

"문자 * 숫자? 돼요! 세 번 반복되게! 근데 문자 + 숫자는 안 돼요~ 에러나요!"

2. 다음 중 두 값이 같은지 비교하는 데 사용되는 연산자는 무엇인가요?

해설:

`==`는 두 값의 동등성을 비교하는 연산자입니다. `=`는 할당 연산자로 값을 변수에 넣을 때 사용됩니다.

"같다! 는 이퀄 두 개! 이거 하나면 에러나요~"

3. 다음 파이썬 코드의 실행 결과는 무엇인가요?

result = 15 % 4
print(result)

해설:

`%` 연산자는 나눗셈 후의 나머지를 반환합니다. 15를 4로 나누면 몫은 3이고 나머지는 3이 됩니다.

"퍼센트 연산자는 진짜 자주 나와요! 특히 반복문이랑 같이 쓸 때!"

4. 다음 중 비교 연산자가 아닌 것은 무엇인가요?

해설:

비교 연산자는 두 값의 크기나 관계를 비교하여 참(True) 또는 거짓(False)을 반환합니다. `>` (크다), `<=` (작거나 같다), `!=` (같지 않다) 등이 비교 연산자입니다. `=`는 값을 할당하는 할당 연산자입니다.

"헷갈리지 마! 하나짜리 이퀄은 넣는 거고, 두 개짜리는 비교하는 거야!"

5. 다음 파이썬 코드의 실행 결과는 무엇인가요?

print("Hello" * 3)

해설:

파이썬에서 문자열과 정수를 곱하면 해당 문자열이 정수만큼 반복되어 새로운 문자열을 생성합니다.

"문자열 반복은 곱하기! 문자열끼리는 곱할 수 없어~"

입출력과 친해지기

프로그램은 사용자와 소통해야 합니다. 키보드로 값을 '입력'받고, 화면으로 결과를 '출력'하는 방법을 배워봅시다. 특히 `input()` 함수는 중요한 특징이 있으니 꼭 기억하세요. 파일 입출력도 개념을 이해해 봅시다.

가상 콘솔 체험

>

"input은 다 문자야! 정수 계산하려면 꼭 int()로 싸줘야 돼요~"

입출력 퀴즈

1. 파일 입출력(File I/O) 작업의 올바른 순서는 무엇인가요?

해설:

파일을 사용하기 위해서는 반드시 '열기' → '처리(읽기/쓰기)' → '닫기' 순서를 지켜야 합니다. 파일을 닫지 않으면 내용이 저장되지 않을 수 있습니다.

"파일은 읽고 쓰고 닫아야 돼요. 까먹으면 저장이 안 돼요!"

2. 파이썬의 `input()` 함수가 사용자로부터 값을 입력받을 때, 기본적으로 어떤 자료형으로 반환하나요?

해설:

`input()` 함수는 사용자가 입력한 모든 값을 기본적으로 '문자열(str)' 형태로 반환합니다. 따라서 숫자로 계산하고 싶다면 `int()`나 `float()` 등으로 형 변환을 해주어야 합니다.

"input은 무조건 문자열이야! 숫자 쓰고 싶으면 꼭 바꿔줘야 돼!"

3. 다음 파이썬 코드의 실행 결과로 올바른 것은 무엇인가요?

name = input("이름을 입력하세요: ")
print("환영합니다, " + name + "님!")

해설:

`input()` 함수는 괄호 안의 문자열을 먼저 출력하고 사용자 입력을 기다립니다. 이후 `print()` 함수에서 입력받은 값을 문자열 연결(`+`)을 통해 함께 출력합니다.

"input은 먼저 물어보고, print는 나중에 대답하는 거야~"

4. 다음 중 파일 쓰기 모드를 나타내는 문자는 무엇인가요?

해설:

`'w'` 모드는 파일을 '쓰기(write)' 모드로 엽니다. 만약 파일이 이미 존재하면 내용을 모두 지우고 새로 쓰며, 파일이 없으면 새로 생성합니다. `'r'`은 읽기, `'a'`는 이어쓰기, `'x'`는 파일이 없을 때만 생성하여 쓰기입니다.

"파일 쓰기는 '더블유'! 기존 파일은 날아갈 수 있으니 조심해~"

5. 파이썬에서 파일 작업 완료 후 `file.close()`를 호출해야 하는 가장 중요한 이유는 무엇인가요?

해설:

파일을 닫지 않으면 파일에 쓴 내용이 제대로 저장되지 않거나, 다른 프로그램이 해당 파일에 접근하지 못하는 문제가 발생할 수 있습니다. `close()`는 버퍼에 남아있는 데이터를 실제로 파일에 쓰고, 시스템 리소스를 반환하는 중요한 역할을 합니다.

"파일 닫는 거 까먹으면 큰일 나! 저장 안 될 수도 있고, 다른 프로그램도 못 써!"

리스트 정복: 여러 데이터 관리

데이터가 많아지면 변수 하나로는 부족해요. 리스트를 사용하면 여러 개의 데이터를 하나의 묶음으로 효율적으로 관리할 수 있습니다. 1차원 리스트부터 행과 열 구조를 가진 2차원 리스트까지, 직접 조작해보며 마스터해봅시다.

리스트 시각화 및 조작 도구

"인덱스는 무조건 0부터 시작! 2차원 리스트는 진짜 중요해요~"

리스트 퀴즈

1. 다음 2차원 리스트 `menu`에서 '튀김'의 가격인 2000을 출력하기 위한 코드로 올바른 것은 무엇인가요?

menu = [["라볶이", 4000], ["순대", 3000], ["튀김", 2000]]

해설:

리스트의 인덱스는 0부터 시작합니다. `menu[2]`는 `["튀김", 2000]` 리스트를 가리키고, 그 리스트의 두 번째 요소인 인덱스 `[1]`이 가격 2000을 정확히 선택합니다.

"여기서 실기 나오면, 인덱스 하나라도 헷갈리면 틀려요~ 조심!"

2. 다음 파이썬 코드의 실행 결과는 무엇인가요?

my_list = [10, 20, 30, 40, 50]
print(my_list[1:4])

해설:

슬라이싱 `[시작:끝]`은 시작 인덱스부터 끝 인덱스 '바로 앞'까지의 요소를 포함합니다. `my_list[1:4]`는 인덱스 1부터 3까지의 요소, 즉 20, 30, 40을 포함합니다.

"슬라이싱 할 땐 뒤에 숫자는 포함 안 돼요! 기억해!"

3. 다음 코드를 실행했을 때, `my_list`의 최종 상태는 무엇인가요?

my_list = ["apple", "banana", "cherry"]
my_list.append("orange")
my_list.remove("banana")

해설:

`append()`는 리스트의 맨 뒤에 요소를 추가하고, `remove()`는 지정된 값을 가진 첫 번째 요소를 제거합니다. 따라서 "orange"가 추가된 후 "banana"가 제거됩니다.

"추가는 append! 삭제는 remove! 리스트 조작은 기본이야~"

4. 다음 중 리스트의 길이를 반환하는 파이썬 함수는 무엇인가요?

해설:

파이썬에서 리스트(또는 문자열, 튜플 등)의 길이를 반환하는 내장 함수는 `len()`입니다.

"len은 리스트 길이! for문 돌릴 때 진짜 많이 써~"

5. 2차원 리스트 `board = [['X', 'O'], ['O', 'X']]`에서 첫 번째 행의 'O'를 출력하기 위한 코드는 무엇인가요?

해설:

2차원 리스트는 `[행 인덱스][열 인덱스]` 순서로 접근합니다. 첫 번째 행은 인덱스 `0`이고, 그 안에서 'O'는 두 번째 요소이므로 열 인덱스 `1`입니다.

"2차원 리스트는 [앞][뒤]! 앞은 행, 뒤는 열이라고 생각하면 돼~"

제어 구조: 프로그램의 흐름 만들기

프로그램을 똑똑하게 만들려면 상황에 따라 다른 행동을 하거나, 특정 작업을 반복하게 해야 합니다. `if`문과 `while`문은 프로그램의 흐름을 제어하는 가장 중요한 도구입니다. 이 개념은 앞으로 배울 모든 것의 기초가 됩니다.

조건문 (if)

'만약 ~라면'처럼 특정 조건이 맞을 때만 코드를 실행합니다.

score = 95
if score >= 90:
    print("A학점 입니다!")
elif score >= 80:
    print("B학점 입니다.")
else:
    print("노력이 필요해요.")

1. 위 코드의 실행 결과는 무엇인가요?

해설:

`score`가 95이므로 첫 번째 조건 `score >= 90`이 참(True)이 되어 "A학점 입니다!"가 출력됩니다.

2. 다음 파이썬 코드의 실행 결과는 무엇인가요?

temp = 25
if temp > 30:
    print("너무 더워요!")
elif temp < 20:
    print("쌀쌀해요!")
else:
    print("적당한 온도입니다.")

해설:

`temp`가 25이므로 첫 번째 조건 `temp > 30`은 거짓입니다. 두 번째 조건 `temp < 20`도 거짓입니다. 따라서 `else` 블록의 "적당한 온도입니다."가 출력됩니다.

"조건문은 순서대로 검사해! 다 아니면 else가 실행되는 거야~"

반복문 (while)

'~하는 동안'처럼 특정 조건이 만족되는 동안 코드를 계속 반복합니다.

count = 0
while count < 3:
    print("반복!")
    count += 1

1. 위 코드는 '반복!'을 몇 번 출력할까요?

해설:

`count`가 0, 1, 2일 때 조건 `count < 3`이 참이므로 '반복!'을 출력하고 `count`가 1씩 증가합니다. `count`가 3이 되면 조건이 거짓이 되어 반복이 종료됩니다. 따라서 총 3번 출력됩니다.

2. 다음 파이썬 코드의 실행 결과로 올바른 것은 무엇인가요?

num = 5
while num > 0:
    print(num)
    num -= 2

해설:

`num`은 5에서 시작하여 2씩 감소합니다. `num > 0` 조건이 참일 때마다 `print(num)`이 실행됩니다. 따라서 5, 3, 1이 출력되고 `num`이 -1이 되면 조건이 거짓이 되어 반복문이 종료됩니다.

"while문은 조건이 True일 때만 돌아! False되면 바로 멈춰!"

반복문 (for)

'~마다'처럼 특정 범위의 항목을 하나씩 꺼내면서 코드를 반복합니다.

fruits = ["사과", "바나나", "체리"]
for fruit in fruits:
    print(fruit)

1. 위 코드의 실행 결과로 올바른 것은 무엇인가요?

해설:

`for` 반복문은 리스트나 문자열과 같은 '반복 가능한(iterable)' 객체의 요소를 하나씩 가져와 실행합니다. `print()` 함수는 기본적으로 각 항목을 새 줄에 출력합니다.

"for문은 순서대로 하나씩! print는 한 줄씩 기억해둬요~"

2. 다음 파이썬 코드의 실행 결과는 무엇인가요?

for i in range(3):
    print(i)

해설:

`range(N)` 함수는 0부터 `N-1`까지의 정수를 순서대로 생성합니다. 따라서 `range(3)`은 0, 1, 2를 생성하고, 각 숫자가 한 줄씩 출력됩니다.

"range는 0부터 시작해서 마지막 숫자 직전까지! 이것도 시험에 잘 나와~"

"제어 구조가 문법에서는 가장 핵심이 될 것 같아. 이거 헷갈리면 뒤에 다 꼬여요~"

지금까지 배운 내용들을 바탕으로 수행평가에서 나올 수 있는 핵심적인 지필 및 프로그래밍 문제 유형을 최종 정리해 드립니다. 각 포인트를 다시 한번 짚어보며 시험에 완벽하게 대비하세요!

지필 평가 예상 문제 유형

  • 변수와 리스트의 차이점 및 사용 목적 설명: 데이터를 한 개 저장하는 변수와 여러 값을 저장하는 리스트의 개념, 그리고 왜 리스트가 필요한지 설명할 수 있어야 합니다.
  • 파이썬의 특징과 인터프리터 방식 설명: 파이썬이 문법이 간단하고 활용성이 높은 이유, 그리고 코드를 한 줄씩 번역/실행하는 인터프리터 방식의 특징을 설명할 수 있어야 합니다.
  • 문자열 연산과 형 변환의 중요성 설명: 문자열 덧셈(`+`), 곱셈(`*`)의 특징, 그리고 `input()` 함수로 받은 문자열을 `int()`나 `float()`으로 형 변환해야 하는 이유를 설명할 수 있어야 합니다.
  • 파일 입출력의 3단계 설명: 파일 열기, 처리, 닫기의 중요성과 각 단계의 의미를 설명할 수 있어야 합니다.

프로그래밍 실기 예상 문제 유형

  • 2차원 리스트 활용: 주어진 2차원 리스트에서 특정 요소(예: 메뉴의 가격)를 인덱싱하여 출력하는 코드를 작성할 수 있어야 합니다. 변수명 스펠링 주의!
  • 사용자 입력, 연산, 형 변환 활용: `input()` 함수로 사용자로부터 값을 입력받고, `int()` 또는 `float()`로 형 변환하여 사칙연산(`+`, `*`, `//`, `%`, `**`)을 수행한 후 결과를 출력하는 코드를 작성할 수 있어야 합니다. (예: BMI 계산기, 과목 평균 구하기)
  • 파일 입출력 코딩: 파일에 특정 내용을 쓰고(write), 닫은(close) 후, 다시 파일을 열어(open) 내용을 읽어(read) 화면에 출력하는 코드를 작성할 수 있어야 합니다.
  • 리스트 조작 (추가/삭제/수정): 리스트에 요소를 추가(`append`), 삭제(`remove`), 특정 인덱스의 값을 수정하는 코드를 작성할 수 있어야 합니다.
  • 제어 구조 활용: `if`문(조건문)과 `while`문(반복문)을 활용하여 특정 조건에 따라 다른 코드를 실행하거나, 반복 작업을 수행하는 간단한 코드를 작성할 수 있어야 합니다. (예: 점수에 따른 학점 출력, 반복 횟수 제어)
  • 2차원 리스트로 이미지 표현 (픽셀 아트): 0과 1로 구성된 2차원 리스트를 사용하여 간단한 이미지(예: 하트)를 표현하는 코드를 이해하고 작성할 수 있어야 합니다.
