프로그래밍 방식 비교

우리가 접하는 프로그래밍 방식은 크게 두 가지로 나눌 수 있습니다. 각각 어떤 특징이 있는지 비교해 봅시다.

블록 프로그래밍

엔트리, 스크래치처럼 블록을 드래그해서 시각적으로 프로그램을 만드는 방식입니다.

👍 장점

  • 코딩 개념을 쉽고 직관적으로 이해
  • 문법 오류가 거의 없음
  • 프로그램 흐름을 시각적으로 파악하기 용이

👎 단점

  • 만들 수 있는 기능이 제한적
  • 전문적인 프로그래밍에는 활용하기 어려움

텍스트 프로그래밍 (파이썬)

파이썬, C언어처럼 코드를 직접 타이핑하여 프로그램을 만드는 방식입니다.

👍 장점

  • 상상하는 모든 프로그램을 만들 수 있음
  • 컴퓨터의 동작을 세밀하게 제어 가능
  • 실제 산업 현장에서 사용하는 전문 기술

👎 단점

  • 초기 문법 학습이 필요함
  • 작은 오타에도 오류가 발생할 수 있음

"쉬우면 쉬울수록 만들 수 있는 건 한계가 있어요~ 어려운 건 어렵지만 뭐든 가능해요!"

왜 파이썬일까?

수많은 텍스트 언어 중 파이썬이 주목받는 이유를 알아봅시다.

✍️

간단한 문법

사람의 언어와 비슷해 배우기 쉽고 코드가 간결해요.

🎁

무료

누구나 자유롭게 사용할 수 있는 오픈소스 언어입니다.

⚡️

즉시 결과 확인

코드를 쓰고 바로 실행하여 결과를 볼 수 있어요.

📚

풍부한 라이브러리

미리 만들어진 코드(도구)가 많아 활용성이 높아요.

언어 처리 방식: 인터프리터 vs 컴파일러

우리가 쓴 코드는 컴퓨터가 이해할 수 있는 언어로 '번역'이 필요합니다. 번역 방식의 차이를 알아봅시다.

"파이썬은 인터프리터예요! 즉, 바로바로 실행 가능하다는 말이에요~"

인터프리터 (Interpreter)

코드를 한 줄씩 읽으며 바로 번역하고 실행하는 '동시통역' 방식입니다. (예: 파이썬)

  • ✔️오류 수정이 쉽고 개발이 유연합니다.
  • ❌컴파일러 방식보다 실행 속도가 느릴 수 있습니다.

컴파일러 (Compiler)

전체 코드를 한 번에 번역해서 실행 파일을 만든 후 실행하는 '전체번역' 방식입니다. (예: C언어)

  • ✔️실행 속도가 매우 빠릅니다.
  • ❌오류를 찾기 어렵고 수정 후 다시 번역해야 합니다.

5월 28일: 코딩의 진짜 시작 - 변수와 연산자

오늘은 프로그램의 가장 기본적인 구성 요소인 '변수'와 데이터를 다루는 '연산자'에 대해 깊이 있게 알아봅니다. 메모리 개념부터 사칙연산, 논리 연산까지 직접 실습하며 코딩의 기초를 탄탄히 다져보세요.

"얘들아~ 오늘부터 진짜 코딩 시작이야! 이거 헷갈리면 뒤에 while if 다 꼬여요~"

변수란 무엇인가?

변수는 데이터를 저장하는 공간이자, 마치 '이름 붙인 상자'와 같습니다.

변수: 데이터를 저장하는 '이름 붙인 상자'

프로그램에서 자주 사용하거나 변동될 수 있는 데이터를 메모리에 저장해 놓고, 그 데이터를 이름으로 불러서 사용하기 위해 변수를 사용합니다. 복잡한 메모리 주소 대신 사람이 기억하기 쉬운 '이름'으로 데이터를 관리하는 것이죠.

"이름만 기억하면 값은 자동! 변수 없으면 사람 뇌가 터져요~"

변수 이름 붙이기 규칙

  • 반드시 영문자로 시작해야 합니다.
  • 숫자는 변수 이름의 뒤에만 붙일 수 있습니다. (예: `a1`, `num2`는 가능, `1a`, `2num`은 불가능)
  • 특수문자(언더스코어 `_` 제외), 띄어쓰기는 사용할 수 없습니다.

"숫자로 시작하면? 파이썬은 계산식인 줄 안다니까~"

변수와 메모리의 관계

컴퓨터는 모든 데이터를 **메모리(RAM)**에 저장합니다. 하지만 메모리 주소는 복잡한 숫자로 이루어져 있어 사람이 외우기 어렵습니다. 변수는 이 메모리 주소에 사람이 이해하기 쉬운 '이름'을 붙여주는 역할을 합니다.

사칙연산과 연산자

데이터를 가지고 다양한 계산을 하기 위해 '연산자'를 사용합니다.

  • `+`: 더하기
  • `-`: 빼기
  • `*`: 곱하기
  • `/`: 나누기 (결과는 실수)

특수 연산자 (중요!)

  • `//`: 몫만 구함 (정수)
  • `%`: 나머지 구함
  • `**`: 제곱 (a**b = a의 b제곱)

두 값이 '크다', '작다', '같다' 등을 비교하여 참(True) 또는 거짓(False)을 반환합니다.

  • `>`: 크다
  • `<`: 작다
  • `>=`: 크거나 같다
  • `<=`: 작거나 같다
  • `==`: 같다 (⚠️ 등호 두 개! `=`는 값을 넣는 것!)
  • `!=`: 같지 않다

예시:

`a = 9`, `b = 4` 일 때

`a > 10` 의 결과: False

`a == 9` 의 결과: True

여러 조건을 조합하여 논리적인 판단을 할 때 사용하며, 결과는 참(True) 또는 거짓(False)입니다.

  • `and`: 그리고 (모든 조건이 참일 때만 True)
  • `or`: 또는 (하나라도 조건이 참이면 True)

"논리는 일상에서도 쓰는 거랑 똑같아요~ 조건문 if에 들어가면 여기부터 쓰게 돼요~"

예시:

`a = 9`, `b = 4` 일 때

`a > 10 and b > 3` 의 결과: False

`a < 10 or b < 2` 의 결과: True

5월 29일: 변수 활용 심화 & 표준 입출력

변수에 숫자 외에 문자도 넣는 방법, 문자열을 다루는 연산자, 그리고 사용자로부터 입력을 받고 결과를 출력하는 표준 입출력 방법을 자세히 배워봅니다.

"얘들아~ 변수에 숫자만 넣는 줄 알았지? 문자도 넣고 더하고 곱하고 다 된다~"

변수 활용 심화: 문자열 연산

변수에는 숫자만 넣는 것이 아닙니다! 문자(텍스트)도 넣고 연산할 수 있어요.

문자열 연산의 종류

  • `+`: 문자열 이어붙이기 (Concatenation)

    예: `"안녕" + "하세요"` → `"안녕하세요"`

  • `*`: 문자열 반복하기 (Repetition)

    예: `"안녕" * 3` → `"안녕안녕안녕"`

"문자 * 숫자? 돼요! 세 번 반복되게! 근데 문자 + 숫자는 안 돼요~ 에러나요!"

"곱하기 먼저 하니까 괄호 꼭! 괄호 안 치면 '나는'만 3번 반복돼요~"

복합 대입 연산자

변수 값에 연산을 더해서 다시 그 변수에 저장하는 편리한 방식입니다.

  • `a += 3` → `a = a + 3` 와 같습니다.
  • `b *= 2` → `b = b * 2` 와 같습니다.

"얘들아, 이건 실기 문제로도 나와요! '복합'이란 단어 나오면 이거예요~"

표준 입출력 (Input/Output)

사용자로부터 입력을 받고, 결과를 화면에 출력하는 방법을 배워봅시다.

입력: `input()` 함수

`input()` 함수를 사용하여 사용자로부터 값을 입력받을 수 있습니다. **주의할 점은 `input()`으로 입력받은 값은 항상 문자(str) 형태로 받아진다는 것입니다.**

  • `a = input("이름을 입력하세요: ")`
  • `num = input("숫자를 입력하세요: ")` (※ 이 상태로 `num`을 숫자로 계산하면 오류!)

"input은 다 문자야! 정수 계산하려면 꼭 int()로 싸줘야 돼요~"

형변환 함수

`input()`으로 받은 문자열을 숫자로 계산하고 싶을 때는 '형변환'을 해야 합니다.

  • `int()`: 문자열을 정수형(Integer)으로 변환합니다. (예: `int("123")` → `123`)
  • `float()`: 문자열을 실수형(Float)으로 변환합니다. (예: `float("3.14")` → `3.14`)

출력: `print()` 함수

`print()` 함수를 사용하여 프로그램의 결과를 화면에 출력합니다. 여러 값을 출력할 때는 콤마(`,`)로 구분합니다.

"문자랑 숫자랑 같이 출력할 땐 콤마로 구분해줘요! + 쓰면 에러 나요!"

하드코딩 vs 입력 기반 프로그래밍

더 유연하고 효율적인 프로그램을 만들기 위한 핵심 개념입니다.

하드코딩 (Hard Coding)

코드 안에 숫자나 문자를 직접 고정시켜 넣는 방식입니다. 예를 들어, `result = 3 + 4` 처럼 값을 직접 지정하는 것이죠. 이는 프로그램이 특정 값에 종속되어 유연성이 떨어집니다. 값이 바뀔 때마다 코드를 직접 수정해야 합니다.

입력 기반 프로그래밍 (Input-based Programming)

사용자로부터 값을 입력받아 그 값에 따라 계산하거나 동작하는 방식입니다. `num1 = input("첫 번째 숫자를 입력하세요: ")` 처럼 사용자에게 값을 입력받으면, 프로그램은 더 유연해지고 다양한 상황에 대처할 수 있습니다.

"계산기 만들 건데 3+4밖에 못 해? 그럼 계산기 아니지~"

파일 입출력 기초

디스크에 저장된 파일을 읽고 쓰는 방식, 영구적인 데이터 저장의 시작입니다.

지금까지는 프로그램이 종료되면 데이터가 사라지는 '표준 입출력(키보드/모니터)'을 사용했습니다. '파일 입출력'은 데이터를 하드 디스크 같은 저장 장치에 영구적으로 저장하고, 필요할 때 다시 읽어오는 방식입니다.

"파일은 읽고 쓰고 닫아야 돼요. 까먹으면 저장이 안 돼요!"

파일 입출력의 3단계

  1. **파일 열기 (`open()`):** 어떤 파일을 어떤 목적으로(읽기/쓰기/추가) 사용할지 지정합니다.
  2. **작업 (`write()`, `read()`, `readline()`, `readlines()` 등):** 파일에 내용을 쓰거나, 파일에서 내용을 읽어옵니다.
  3. **파일 닫기 (`close()`):** 작업이 끝난 파일을 안전하게 닫습니다. (매우 중요!)

주요 파일 관련 함수

`f = open("파일명.txt", "모드")`

파일을 열고 파일 객체를 반환합니다.

  • `"r"`: 읽기 모드 (Read) - 파일이 없으면 오류 발생
  • `"w"`: 쓰기 모드 (Write) - 파일이 없으면 새로 생성, 있으면 기존 내용 덮어씀
  • `"a"`: 추가 모드 (Append) - 파일이 없으면 새로 생성, 있으면 기존 내용 뒤에 추가

`f.write("내용")`

파일에 문자열을 씁니다. `print()`와 비슷하지만 대상이 모니터가 아닌 파일입니다.

"write는 파일에 적는 거예요. print랑 비슷하지만 대상이 파일임~"

`f.close()`

열었던 파일을 닫습니다. 변경사항 저장 및 자원 해제를 위해 필수입니다.

`f.read()`

파일 전체 내용을 하나의 문자열로 읽어옵니다.

`f.readline()`

파일에서 한 줄씩 읽어옵니다.

`f.readlines()`

파일의 모든 줄을 읽어와 리스트 형태로 반환합니다.

오늘 배운 내용을 잘 이해했는지 간단한 퀴즈로 확인해 보세요!

Q. 코드에 숫자나 문자를 직접 고정시켜 넣는 방식을 무엇이라고 할까요? (이 방식의 단점은 무엇인가요?)

A. 하드코딩. 값이 바뀔 때마다 코드를 직접 수정해야 하는 단점이 있습니다.

Q. 파일 입출력을 할 때 필요한 3단계는 무엇일까요?

A. 파일 열기 → 작업 (쓰기/읽기) → 파일 닫기

Q. 파일 쓰기 모드, 읽기 모드, 추가 모드를 나타내는 각각의 알파벳은 무엇일까요?

A. 쓰기: `"w"`, 읽기: `"r"`, 추가: `"a"`

Q. 파일에 내용을 쓰는 함수는 `f.write("내용")` 입니다. 이 함수는 `print()` 함수와 무엇이 다를까요?

A. `print()`는 모니터에 출력하지만, `f.write()`는 파일에 내용을 저장합니다.

Q. 평균 계산기나 BMI 계산기처럼 사용자로부터 값을 입력받아 처리하는 것이 하드코딩보다 왜 더 좋은 방식일까요?

A. 사용자 입력에 따라 유연하게 계산할 수 있어, 값을 바꿀 때마다 코드를 수정할 필요가 없기 때문입니다.

6월 9일: 리스트와 2차원 데이터 구조

변수의 한계를 넘어, 여러 데이터를 효율적으로 저장하고 관리하는 '리스트'를 배워봅시다. 특히 실기 시험에 자주 등장하는 2차원 리스트를 집중적으로 다룹니다.

"얘들아~ 이건 진짜 실기 시험에도 나와요~"

변수와 리스트 차이점

변수는 데이터 한 개, 리스트는 여러 데이터를 저장합니다.

**변수**는 데이터를 **한 개**만 저장할 수 있습니다. 예를 들어 31명의 이름을 저장하려면 변수 31개를 만들어야 하는 비효율적인 상황이 발생합니다. 이를 해결하기 위해 등장한 개념이 바로 **리스트**입니다.

리스트 예시: `my_list = ["봄", "여름", "가을", "겨울"]`

  • **대괄호(`[]`)**를 사용합니다.
  • 여러 개의 값을 순서대로 저장할 수 있습니다.

리스트: 인덱싱, 슬라이싱, 수정

리스트에 저장된 값들은 '인덱스'라는 고유한 번호를 가집니다. **인덱스는 무조건 0부터 시작합니다!**

"여기서 실기 나오면, 인덱스 하나라도 헷갈리면 틀려요~ 조심!"

인덱싱 (Indexing): 특정 위치의 한 요소 가져오기

리스트: ["봄", "여름", "가을", "겨울"]

결과:

슬라이싱 (Slicing): 범위로 잘라서 여러 요소 가져오기

`my_list[시작_인덱스:끝_인덱스]` 형태로 사용하며, **끝 인덱스의 요소는 포함되지 않습니다!** 시작 인덱스를 생략하면 처음부터, 끝 인덱스를 생략하면 끝까지 가져옵니다.

리스트: ["봄", "여름", "가을", "겨울"]

리스트 요소 수정

인덱스를 이용하여 특정 위치의 값을 다른 값으로 '덮어쓰기' 할 수 있습니다.

현재 리스트: ["봄", "여름", "가을", "겨울"]

2차원 리스트 (Nested Lists)

리스트 안에 또 다른 리스트가 들어있는 형태로, 마치 표(행과 열)처럼 데이터를 구조화하여 저장할 때 사용합니다. 복잡한 데이터를 체계적으로 관리하는 데 매우 유용합니다. (예: 구구단 전체, 메뉴판 등)

"이게 진짜 실기 시험에 나온다니까~ 2차원 리스트는 진짜 중요해요~"

2차원 리스트 예시: 메뉴판

`menu = [["라볶이", 4000], ["순대", 3000], ["튀김", 2000]]`

인덱스 (행) 인덱스 (열 0: 이름) 인덱스 (열 1: 가격)
2차원 리스트에서 값 가져오기

`list[행_인덱스][열_인덱스]` 형식으로 접근합니다.

결과:

"문제에 변수명 떡볶이재료 나오면 그 spelling 한 글자라도 틀리면 채점 안 돼요! 조심!"

Q. 리스트에서 특정 요소의 위치를 나타내는 번호를 무엇이라고 하며, 몇 번부터 시작할까요?

A. 인덱스, 0번부터 시작합니다.

Q. `my_list = ["사과", "바나나", "체리", "딸기"]` 일 때, `my_list[1:3]`의 결과는 무엇일까요?

A. `["바나나", "체리"]` (끝 인덱스 3은 포함되지 않습니다.)

Q. `menu = [["라면", 3000], ["김밥", 2000]]` 이라는 2차원 리스트에서 '김밥'의 가격인 `2000`을 가져오려면 어떻게 접근해야 할까요?

A. `menu[1][1]` (두 번째 행의 두 번째 열)
